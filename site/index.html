Pricing question

How should return-shipping insurance be priced for e-commerce merchants?

This project builds an exposure-level dynamic pricing workflow for e-commerce return-shipping insurance, linking logistics, returns, covered claims, product risk, and seller history. GLM models frequency and severity into explainable rate factors; XGBoost challenges the risk ranking; seller credibility, loss-ratio backtesting, stress testing, and pricing guardrails simulate an insurer's launch and monitoring process.

Method

A complete pricing workflow, not a single model score.

  1. 01

    Define exposure as order_id + order_item_id + seller_id + product_id.

  2. 02

    Separate return behavior from covered insurance claims.

  3. 03

    Estimate pure premium with frequency × severity using quote-time features.

  4. 04

    Apply seller credibility, monitor loss ratio, stress test adverse scenarios.

  5. 05

    Use XGBoost as a challenger score, not an automatic replacement for GLM.

Core results

Pricing, credibility, and monitoring work together.

GLM produces the explainable expected-loss rate basis; seller credibility shrinks sparse merchant experience toward that base; XGBoost is tested as a challenger risk-ranking model and kept out of the final rate basis when calibration is weaker. Backtesting and stress testing turn the rate model into an operating control loop.
